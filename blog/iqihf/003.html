Interview Questions I've Failed, 003: Vanilla JS

I use jQuery on most front end projects, and as a result, I sometimes have trouble manipulating the DOM with vanilla JS. It's occasionally asked that you write a bit of code that doesn't rely on a library like jQuery, so here's a refresher/reference/primer on what native methods you can use and how to implement common jQuery methods.

Selecting Elements

First off, let's go through the ways we can select elements (el is shorthand for element, FYI): 

//

// select all elements that match the passed selector(s)
let el = document.querySelectorAll('.el');

// select just the first matching element
let el = document.querySelector('.el p');

// select a single element by id
let el = document.getElementById('el');

// select multiple elements by tag name
let el = document.getElementsByTagName('div');

// select multiple elements by class name
let el = document.getElementsByClassName('p');

// We can also check out the parents and children
// of any element we select
let children = el.children;

Adding & Removing Elements

Of course, we won't just be working with existing elements, we'll most likely want to add new elements and maybe remove them. Unlike setting the visiblity or display property of an element, when we use these methods, we'll actually by adding or removing these elements from the DOM altogether. 

// First off, let's create a new element
let el = document.createElement('div');

// And grab a parent element
let parentEl = document.getElementByTagName('body');

// This element hasn't actually been placed into the DOM,
// so let's do that:

// Add a new element after all of the existing children elements
parentEl.appendChild(el)

// Add a new element before all of the existing children elements
// Note – referenceNode is an existing child element of the parent
// element of your choosing. insertBefore will insert the new element
// right before the referenceNode element. You can pass null to insert
// at the end of all the children.
parentEl.insertBefore(el, referenceNode)

// Here's how to remove an element, it's a bit weird, you first have
// select the element, reference it's parent, and then remove the element
let elToDelete = getElementById('element-id');
elToDelete.parentNode.removeChild(elToDelete);

Working With Classes

A common thing to do with a selected element is to use a jQuery method like addClass() or removeClass(). Here's how we do that with vanilla JS: 

// add a new class
el.classList.add('class-name');

// Remove an existing class
el.classList.remove('class-name');

// Toggle the existence of a class
el.classList.toggle('class-name');

// check to see if an element has a particular class
el.classList.contains('class-name');

Event Listeners

When building User Interfaces, you'll often need to "listen" for particular user behavior, especially when building custom UI controls like buttons, tabs, accordions, etc. 

// html:
// 

// get the element
let el = document.querySelector('.btn');

// can listen to any of these types:
// https://developer.mozilla.org/en-US/docs/Web/Events
// also accepts a callback function to run after event takes place
el.addEventListener('click', function() {
  // code to run when the button is clicked…
})

// you can also remove event listeners
el.removeEventListener('click', listenerFunction);

// A common thing to do in jQuery is to run code when the document
// is ready (the page has loaded). Here's how to do it in vanilla JS
const ready = (fn) => {
  // check to see if the page is still loading
  if (document.readyState !== 'loading') {
    // if the page has loaded, run the supplied function
    fn();
  } else {
    // otherwise, listen for the 'DOMContentLoaded' event and run the
    // supplied function when it happens
    document.addEventListener('DOMContentLoaded', fn);
  }
}
